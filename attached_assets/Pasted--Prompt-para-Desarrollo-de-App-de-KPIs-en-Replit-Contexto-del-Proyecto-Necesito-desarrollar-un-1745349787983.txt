# Prompt para Desarrollo de App de KPIs en Replit

## Contexto del Proyecto

Necesito desarrollar una aplicación web en Replit que permita a Econova gestionar y dar seguimiento a los KPIs de sus clientes corporativos. La primera versión se enfocará en el seguimiento de KPIs para dos empresas: Dura International y Grupo Orsega, ambas pertenecientes al mismo grupo corporativo en la industria química.

## Requisitos Funcionales

1. **Gestión de Usuarios**:
   - Administradores (personal de Econova) que pueden ver y editar todos los KPIs
   - Visualizadores (clientes) que solo pueden ver sus propios KPIs
   - Autenticación segura

2. **Gestión de Empresas**:
   - Capacidad para administrar múltiples empresas (empezando con Dura International y Grupo Orsega)
   - Cada empresa tiene áreas funcionales definidas

3. **Gestión de KPIs**:
   - Crear, leer, actualizar y eliminar KPIs
   - Cada KPI pertenece a un área específica y a una empresa
   - Almacenamiento de datos históricos para análisis de tendencias

4. **Dashboard Principal**:
   - Visión general de todos los KPIs por empresa
   - Filtros por área, estado, periodo
   - Indicadores visuales de cumplimiento (verde, amarillo, rojo)

5. **Formularios de Entrada**:
   - Interfaces simples para que los administradores ingresen los datos de KPIs
   - Validación de datos

6. **Reportes Automáticos**:
   - Generación de informes PDF/Excel
   - Comparativas entre periodos
   - Visualización de tendencias

7. **Notificaciones**:
   - Alertas por KPIs que no cumplen objetivos
   - Recordatorios para actualización de datos

## Estructura de Datos

### Empresas
- ID
- Nombre
- Descripción
- Sector/Industria
- Logo
- Fecha de creación

### Áreas
- ID
- Nombre
- Descripción
- Empresa ID (relación)

### KPIs
- ID
- Nombre
- Descripción
- Área ID (relación)
- Empresa ID (relación)
- Unidad de medida
- Objetivo
- Frecuencia de medición (semanal, mensual, trimestral, anual)
- Método de cálculo
- Responsable

### Valores de KPIs
- ID
- KPI ID (relación)
- Valor
- Fecha de registro
- Periodo (a qué mes/trimestre/año corresponde)
- Porcentaje de cumplimiento
- Estado (cumple, alerta, no cumple)
- Comentarios/Observaciones

### Planes de Acción
- ID
- KPI ID (relación)
- Descripción del problema
- Acciones correctivas
- Responsable
- Fecha de inicio
- Fecha de finalización prevista
- Estado (pendiente, en proceso, completado)
- Resultados

### Usuarios
- ID
- Nombre
- Email
- Contraseña (hash)
- Rol (admin, viewer)
- Empresa ID (relación, para usuarios tipo cliente)
- Último acceso

## Tecnologías a Utilizar

- **Frontend**: React.js con Tailwind CSS
- **Backend**: Node.js con Express
- **Base de Datos**: MongoDB (NoSQL para flexibilidad en la estructura de datos)
- **Autenticación**: JWT (JSON Web Tokens)
- **Gráficos**: Chart.js o Recharts
- **Generación de Reportes**: jsPDF y ExcelJS
- **Hosting**: Replit

## Diseño de Interfaz

La interfaz debe ser limpia, profesional y enfocada en la visualización de datos. Preferencia por:
- Paleta de colores corporativos (azul, blanco, gris)
- Diseño responsive que funcione en desktop y dispositivos móviles
- Dashboard con tarjetas para cada área y KPI
- Uso de gráficos para visualizar tendencias
- Íconos intuitivos para estados (✓, !, ✗)

## Datos Iniciales para Cargar

### Áreas Funcionales
- Contabilidad y Finanzas
- Compras
- Ventas
- Soporte de Ventas
- Logística
- Almacén
- Tesorería
- Calidad
- Dirección

### KPIs por Área

#### Contabilidad y Finanzas
- Precisión en estados financieros (Objetivo: 99%)
- Velocidad de rotación de cuentas por cobrar (Objetivo: < 45 días)
- Cumplimiento de obligaciones fiscales (Objetivo: 100%)
- Facturación sin errores (Objetivo: 100%)
- Control de costos - margen bruto (Objetivo: > Presupuestado)

#### Compras
- Nivel de inventario óptimo (Objetivo Dura: Máx. 2 meses secantes) (Objetivo Orsega: Máx. 6 ton metilato)
- Tiempo de respuesta a Ventas (Objetivo: Máx. 4 horas)
- Tiempo de entrega de proveedores (Objetivo: 95% cumplimiento)
- Optimización de costos de compra (Objetivo: < 5%)

#### Ventas
- Volumen de ventas alcanzado (Objetivo: 100% de cuota)
- Porcentaje de crecimiento en ventas (Objetivo: +10% vs año anterior)
- Nuevos clientes adquiridos (Objetivo: 2 nuevos/mes)
- Tasa de retención de clientes (Objetivo: 90%)
- Satisfacción interdepartamental (Objetivo: Retroalimentación continua)

#### Soporte de Ventas
- Actualización del CRM (Objetivo: 100%)
- Tiempo de respuesta a solicitudes (Objetivo: < 24 horas)
- Precisión en documentación de órdenes (Objetivo: 98%)
- Eficiencia en reportes de soporte (Objetivo: 100% en plazo)
- Seguimiento a evaluación de muestras (Objetivo: 100%)
- Seguimiento a redes sociales (Objetivo: 2 posts/semana)
- Recuperación de ventas de clientes (Objetivo: % de clientes reactivados)

#### Logística
- Precisión de inventarios (Objetivo: 100%)
- Cumplimiento de tiempos de entrega (Objetivo: 100%)
- Costos de transporte (Objetivo: < Inflación anual)
- Satisfacción de clientes internos (Objetivo: 100%)
- Tiempo de recuperación de evidencias (Objetivo: < 24 horas)

## Consideraciones Adicionales

- La aplicación debe ser escalable para añadir más empresas en el futuro
- Priorizar la seguridad de los datos
- Diseñar pensando en la experiencia de usuario
- Incluir ayuda contextual y tooltips para facilitar el uso
- Implementar un sistema de logs para auditoría
